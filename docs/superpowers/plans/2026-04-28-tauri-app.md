# code-switcher Tauri App Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Tauri v2 desktop app that manages code-switcher profiles on macOS, Windows, and Linux.

**Architecture:** React/TypeScript renders the account console. Rust Tauri commands own all filesystem access and expose redacted profile data. Profile service logic is separated from Tauri command wrappers so it can be unit tested with temporary directories.

**Tech Stack:** Tauri v2, Rust, React, TypeScript, Vite, CSS modules/plain CSS, npm.

---

## File Structure

- `package.json`: frontend scripts and Tauri dev/build commands.
- `index.html`: Vite entry point.
- `src/main.tsx`: React mount.
- `src/App.tsx`: account manager UI.
- `src/App.css`: compact desktop UI styling.
- `src-tauri/Cargo.toml`: Rust dependencies.
- `src-tauri/build.rs`: Tauri build hook.
- `src-tauri/tauri.conf.json`: app metadata and bundle config.
- `src-tauri/capabilities/default.json`: minimal frontend permissions.
- `src-tauri/src/main.rs`: app entry.
- `src-tauri/src/lib.rs`: Tauri command registration.
- `src-tauri/src/profiles.rs`: profile service, JWT parsing, atomic writes, tests.
- `README.md`: add desktop app requirements and commands.
